# 财联社与「24 小时滚动新闻」数据源说明

本文档说明：**能否**获取财联社类实时快讯、**助手侧**能做什么、若在「元精灵」中集成**应如何选型**与合规边界。  
**不涉及**对非授权接口的逆向、模拟客户端或网页爬虫实现。

## 直接结论

- **在 AI 对话里**：不能像 App 一样长期订阅「财联社实时电报」滚动流；只能按需检索公开网页并概括，**不是**持牌行情/快讯数据服务。
- **在软件里**：财联社内容通常来自其**自有后端**；**未见**稳定、公开且条款明确允许第三方商用的官方 Open API 文档（以常见情况为准）。因此：
  - **不应**把未获授权的「扒接口 / 仿客户端」作为产品主方案；
  - 若要做「类财联社」体验，应优先考虑：**官方或数据商授权**、**对方明确允许的 RSS/Open Data**、或**自有/已授权数据源**。

## 常见接入方式（对比）

| 方式 | 说明 |
|------|------|
| 官方/商业数据 | 向财联社或持牌数据商采购，合同与合规清晰，适合正式产品。 |
| RSS / 公开聚合 | 在**版权与条款允许**的前提下轮询 RSS；源地址可能变更，需维护。 |
| 第三方转发 API | 网络上存在封装服务，存在**稳定性、版权、隐私与服务条款**风险，不宜作为默认依赖。 |
| 自写爬虫 | 易触发反爬、违反 ToS 与版权风险，页面结构一变即失效。 |

## 若在「元精灵」中做「快讯滚动」前需明确

1. **数据源**：是否已有**书面或条款允许第三方使用**的 URL/API（含鉴权）？
2. **产品形态**：主窗口侧栏、独立窗口、托盘旁提示？轮询间隔（如 30s/60s）与失败降级？
3. **技术落点**（在合法数据源确定后）：可与现有栈一致，例如在 [`src-tauri`](../src-tauri) 用 `reqwest` 拉 JSON，或由前端 `fetch` + 定时器；须在 **Tauri 能力（capabilities）与域名白名单** 中放行目标域。

## 建议后续方向

- **不限定财联社**：可评估**公开且条款允许**的媒体 RSS 等，做统一的「快讯列表」组件与同一套轮询/缓存策略。
- **必须财联社**：走**官方商务**或对方提供的**合法接入文档**，拿到接口说明后再实现 Tauri 命令与 UI。
